# PRD + approved Definition of Done replace GOAL.md as the intent contract

Loop-engineering references (including our own early drafts) use a `GOAL.md` as the human-owned objective file. We decided against a persistent GOAL.md: the PRD the human already writes **is** the intent contract, and the only derived intent artifact is the **Definition of Done** (`.ai/DoD.md`) — generated by bootstrap from the PRD, approved by the human at the single mandatory gate, then immutable (the engine may propose changes, never apply them).

## Considered Options

- **GOAL.md as a persistent artifact** — rejected: it duplicates the PRD and creates a second intent file that can drift from the first.
- **No human gate at all (full autonomy from PRD drop)** — rejected: the engine would derive the acceptance criteria it grades itself against, so a misread PRD becomes a verified-wrong feature.
- **Human approves the full execution plan** — rejected: the plan is machine-owned by principle; the human approves only *what done means*, never *how to get there*.

## Consequences

- The DoD is the only human-owned file inside the otherwise machine-owned `.ai/`.
- Every run's first Escalation Request is the DoD approval.
- Changing the DoD after approval is a Tier-3 event: propose, stop, wait.
